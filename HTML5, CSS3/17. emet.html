Emmet이란? 강력한 자동완성 기능 등으로 HTML 작성 속도를 크게 향상시켜주는 플러그인 Emmet (에밋)을 사용하면 HTML & CSS의 작업 속도를 엄청나게 향상시킬 수 있다!! 👏🏽👏🏼👏🏻 VS Code는 emmet을 기본적으로 지원하고 Tab키를 통해 사용한다. 🧶 HTML 구조화 하기 (Nesting operators) Emmet의 문법을 활용하여 요소들의 구조를 간편히 생성할 수 있다. Emmet 문법 요소 (Elements) 생성 생성하려는 요소의 이름을 입력한 뒤 tab을 누르면 태그가 자동 생성된다. Emmet에는 미리 정해진 태그 이름이 없다. 즉, 아무 이름으로나 태그를 생성할 수 있다.(에디터에 따라 그렇지 않은 경우도 있다.) html:5[tab] 결과: Document helloworld[tab] 결과: 자식(하위) 요소 > >를 사용하여 자식 요소를 생성할 수 있다. : div>ul>li
형제 요소 + +를 사용하여 한 요소와 같은 단계에 위치한 요소를 생성할 수 있다. div>p+span

올라가기 ^ ^를 사용하여 한 단계 위에 요소를 배치할 수 있다. div>ul>li^p+a

^를 여러번 입력하면 입력한 만큼 올라갈 수 있음 div>ul>li^^div
div+div>p>span+em^bq span+em은 p태그의 하위 단계에 위치하지만 bq는 ^로 인해 p태그와 같은 단계에 위치하게 된다. 결과 :

Use as many ^ as needed to move up more levels. div+div>p>span+em^bq

div+div>p>span+em^^bq

div+div>p>span+em^^^bq

반복하기 * Multiplication: * Use ** * ** operator to define how many times element should be outputted. div>ul>li*3
ul>li*5 …outputs to
ul>li*3 …outputs to
그룹화 () Grouping: () Use () to group subtrees in complex abbreviations. div>(header>ul>li*2)+footer
( 🎈 그룹화를 통해 `footer`는 `header`의 형제 요소가 되었다. ) 만약 괄호 없이 div>header>ul>li*2+footer로 작성하게 된다면?
↘ 🧐 footer가 li의 형제요소가 되어버리니 주의하자 div>(header>ul>li*2>a)+footer>p div>(header>ul>li*2>a)+footer>p …expands to

Each group contains abbreviation subtree and all the following elements are inserted at the same level as the first element of group. It is possible to nest groups inside each other and combine them with ** * ** operator. (div>dl>(dt+dd)*3)+footer>p
